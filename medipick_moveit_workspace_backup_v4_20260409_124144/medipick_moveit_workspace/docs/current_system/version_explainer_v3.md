# V3 版本解释

## 1. V3 是什么

V3 不是简单的“小修小补版本”，而是一次主流程重定义。

它的核心变化是：

- 不再把 `prepare` 理解成一个必须精确对齐的固定点
- 不再默认让手臂一开始就参与所有接近动作
- 不再把进柜动作看成一次自由的全局 pose-goal 规划

V3 把整件事拆成了：

- 底盘先进入工作区
- 升降先进入目标高度带
- 柜口外选一个 `pre-insert`
- 再从 `pre-insert` 进入柜内

## 2. 为什么要从旧版本切到 V3

旧链路最大的问题不是“不够复杂”，而是“很多阶段定义本身就不对”。

典型问题包括：

- 旧 `prepare` 过度追求投影对齐，导致柜外就把问题做死
- 底盘没先拿到安全工作区，手臂却过早参与接近
- 升降被当成普通连续规划自由度使用，但真实机构更像粗调闭环
- 末端是吸盘，不是夹爪，进入柜体更像插入而不是抓握包络
- `pre-insert/final/retreat` 之间存在很多历史 fallback，行为不稳定也不透明

V3 的目标就是把这些错误假设拆掉。

## 3. V3 的核心思想

### 收臂优先

底盘还没进入工作区之前，先收臂。

这解决的是：

- 通行安全
- 底盘运动时与周边物体干涉
- 让“到位”这件事先发生在更低维的底盘层面

### 底盘负责进工作区

底盘不负责精抓取，只负责把机器人带到一个“能工作”的区域。

这意味着：

- `BASE_ENTER_WORKSPACE` 是粗定位
- 不要求这一阶段就把末端对得非常准

### 升降负责进高度带

升降机构真实语义不是高精轨迹轴，而是：

- 升
- 降
- 停
- 实时回读高度

所以 V3 用“高度带”而不是“精确单点”。

### 柜口外 `pre-insert`

V3 把 `pre-insert` 放在柜口外。

这样做的好处是：

- 柜外先选一个相对容易到达的位姿
- 柜内最后那段尽量短、直、受限
- 算法和调试都更可解释

### seeded IK 优先

在 `PLAN_TO_PRE_INSERT` 阶段，V3 当前最成功的改动之一是：

- 优先从当前状态附近找 IK
- 再构造短 joint-space 轨迹
- 最后才退回普通 pose-goal 规划

这显著减少了：

- `r1` 大幅绕圈
- MoveIt 选到奇怪 IK 分支

## 4. V3 当前已经解决了什么

### 模型层

- simple4 模型已经替换为当前运行时主模型
- 坐标前向和上向已修正到项目语义
- `raise_joint=0` 已对齐为升降最低位

### 流程层

- 主状态机已经收成 V3 语义
- `ARM_STOW_SAFE` 变成硬规则
- `LIFT_TO_BAND` 用高度带判定
- `SELECT_PRE_INSERT` 被压成轻量 pose 选择
- `PLAN_TO_PRE_INSERT` 有 seeded IK 优先

### 工具层

- 随机柜体实验脚本已可批量跑
- 实验记录已自动输出参数、结果、阶段耗时
- 失败分析已可自动生成 `analysis.md`

## 5. V3 当前还没有彻底解决什么

### 启动/场景同步

随机实验里已经暴露出：

- `ACQUIRE_TARGET` 阶段仍可能因为 `/compute_fk` 超时和 scene/TF 不稳卡住

这不是主抓取逻辑问题，但它会影响批测稳定性。

### `pre-insert` 还不够自适应

当前为了提速，`pre-insert` 默认被压成：

- 第一候选
- pose-only 选择

这提高了速度，但牺牲了一部分普适性。

### `r1` 限制还是后验拒绝为主

现在已经有 `r1` 阶段运动限幅，但很多时候还是：

- 先规划出来
- 再因为 `r1` 过大被拒

理想的 V4 应该把它更早地并到候选和 IK 终点筛选里。

## 6. V3 的优点

- 主链清晰
- 行为比旧版更可解释
- 任务职责边界更合理
- seeded IK 明显改善了预备位姿阶段的流畅性
- 已经有一套可批量复现和统计的实验机制

## 7. V3 的局限

- 对启动时序和场景同步还不够鲁棒
- `base_standoff` / `pre_insert_offset` 仍偏常数驱动
- `pre-insert` 还没做成更成熟的小候选搜索器
- 撤退与 `r1` 偏好还有继续提稳空间

## 8. 为什么接下来要做 V4

进入 V4 不是因为 V3 完全失败，而是因为 V3 已经把主链“收对了”，现在可以开始处理更像产品化的问题：

- 就绪屏障
- 场景同步稳定性
- 参数自适应
- `r1` 代价前置
- 更有针对性的候选和失败恢复

也就是说：

- V3 解决的是“方向对不对”
- V4 解决的是“批量随机场景下稳不稳”

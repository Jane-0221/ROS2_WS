# Medipick V3 方案：面向货柜吸取任务的重构设计与实施计划

## 1. 文档目的

这份文档是本项目 V3 版任务流程的正式设计稿。它不是对现有代码的小修小补，而是基于当前项目实际约束，对抓取流程、阶段划分、规划责任边界、控制接口和代码结构进行一次完整重构。

V3 的输入来源主要有两部分：

- [v3初始.md](/home/lzoolloozl/projects/medipick_moveit_workspace/docs/v3初始.md) 中关于 `pre-align / insert` 的总体思路
- 当前项目的真实工程约束，包括：
  - 底盘控制不够精准
  - 升降杆不是理想连续关节，而是“升 / 降 / 停”控制，带实时高度回读
  - 高度值本质上是升降杆顶部相对机体内部固定参考的相对偏移
  - 末端执行器是吸盘，不是夹爪
  - 当前旧状态机中的若干阶段定义和职责划分本身就是错误的

这意味着 V3 不能简单照搬通用 MoveIt 教程，也不能继续兼容旧版阶段命名和旧版 fallback 逻辑。V3 必须从任务本质出发重建。

## 2. 对 `v3初始.md` 的判断

`v3初始.md` 的核心方向是对的，但只能作为原则，不能直接作为实现方案。

### 2.1 应保留的核心思想

- 不再把 `prepare` 理解成单个固定点，而是理解成柜口外的一片候选区域
- 将任务拆成“大范围接近”和“受限空间插入”两类动作
- 根据最终吸取位姿，反推一批候选 `pre-insert / pre-align` 位姿
- 最后进入柜体的动作应更短、更直、更受限，而不是继续做完全自由规划
- 候选位姿应进行评价，而不是手写一个固定 heuristics 点

### 2.2 不能直接照搬的部分

- 不能把升降杆当成普通 MoveIt 连续规划关节来依赖
- 不能假设底盘到位精度足够高，因此“先规划一个精确 base pose，再把手伸进去”的假设不可靠
- 不能按夹爪逻辑设计姿态约束和抓取闭环，吸盘更强调法向接触、吸附容差、插入稳定性
- 不建议第一阶段直接上 MoveIt Task Constructor
- 不建议第一版就做 20-50 个重候选并全部走完整 `IK + collision + full OMPL + Cartesian`，那样实现和调参成本太高

结论是：

- `v3初始.md` 的任务分解思想合理
- 但必须针对本项目做“工程降阶”和“接口重定义”

## 3. 旧方案为什么必须推翻

旧流程的根本问题不是参数不好，而是阶段定义与实际执行对象错位。

### 3.1 旧流程的错误前提

- 把底盘、升降、机械臂都当成同类精确执行器处理
- 把 `prepare` 视为一个必须精确达到的几何点
- 把最终进柜动作当作普通 pose goal 规划问题
- 把吸盘末端按“通用末端执行器”处理，而不是按法向吸取工具处理

### 3.2 旧流程带来的直接后果

- `prepare` 阶段被迫承担了过多几何对齐责任
- 一旦底盘有误差，后续阶段都建立在错误初值上
- 升降动作和手臂规划没有形成清晰分工
- 进柜动作经常依赖自由采样规划在狭窄空间中“碰巧找到解”
- 某些旧阶段虽然名字存在，但并不对应真实可执行的控制闭环

### 3.3 V3 的重构原则

- 底盘负责粗定位，不负责精确末端对齐
- 升降负责把目标大致拉入可作业高度带，不负责末端细对齐
- 机械臂负责末端姿态和插入路径
- 吸盘抓取的关键不是“到达一个点”，而是“以合适法向和小横向误差稳定接触目标”
- 柜外与柜内应视为两种规划问题

## 4. V3 的任务本质定义

V3 将整个任务重新定义为：

1. 感知到目标药盒的吸取位姿 `T_pick`
2. 根据货格开口方向，生成一组位于柜口外的候选 `T_pre_insert`
3. 通过底盘粗定位和升降粗调，把机器人带入“手臂可插入工作区”
4. 通过车臂联合规划到达最优 `T_pre_insert`
5. 从 `T_pre_insert` 做短距离、近似直线、姿态受限的插入与吸取
6. 吸取后按安全方向撤出，再离柜恢复

V3 的本质不是“多规划一次”，而是把以前混在一起的问题拆成三个不同层次：

- 粗定位：底盘 + 升降
- 对齐：车臂联合到柜口外候选位姿
- 插入：短程、强约束、以吸盘法向为中心的末端推进

## 5. V3 的系统约束模型

### 5.1 底盘约束

- 底盘存在横向、角度和停止位置误差
- 底盘不适合承担“最后几厘米 / 几度”的精确对齐职责
- 因此底盘目标只应定义为“进入工作区”，而不是“末端已对齐”

设计结论：

- 底盘阶段的完成条件使用较宽松容差
- 底盘到位后应保留手臂补偿空间
- 候选 `pre-insert` 的打分必须考虑对底盘误差的鲁棒性

### 5.2 升降杆约束

- 升降杆控制接口是 `升 / 降 / 停`
- 同时可实时回读当前高度
- 高度量是相对量，而不是一个严格全局绝对位姿
- 升降杆更像一个低带宽、单自由度的粗调机构

设计结论：

- V3 中升降杆不作为自由规划关节参与复杂局部插入
- 升降阶段独立于手臂精细姿态规划
- V3 需要维护“目标高度带”而不是单一理想高度
- 控制逻辑应支持：
  - 目标高度上方则下行
  - 目标高度下方则上行
  - 进入容差带后停止
- 阶段成功条件来自实时回读，而不是轨迹想象值

### 5.3 吸盘约束

- 吸盘不是夹爪，没有闭合包络动作
- 吸附成功依赖法向接触、姿态稳定和接触面质量
- 相比夹爪，吸盘更怕横向偏差和接触时姿态漂移

设计结论：

- 最终位姿的核心约束是“工具轴与目标法向关系”
- `insert` 阶段要尽量直线、低横摆、低姿态漂移
- `prepare` 候选的优先级不是最像最终位姿，而是最容易稳定插入并吸附

## 6. V3 新阶段定义

V3 不继续沿用旧阶段语义，而采用以下正式阶段。

### 6.1 `ACQUIRE_TARGET`

职责：

- 接收目标药盒吸取位姿 `T_pick`
- 获取货格进入方向 `n_entry`
- 生成柜体入口平面、层板边界、可插入深度等局部场景语义

输出：

- `TaskGeometry`
  - `pick_pose`
  - `entry_direction`
  - `entry_plane`
  - `cabinet_bounds`
  - `target_height_hint`
  - `insert_depth`

### 6.2 `ARM_STOW_SAFE`

职责：

- 将机械臂收至不会妨碍底盘和升降调整的安全姿态
- 这一阶段不追求最终抓取便利，只保证后续粗定位安全

说明：

- 这是 V3 唯一保留的“固定模板姿态”阶段
- 它的意义是缩小系统状态空间，而不是做对齐
- 底盘进入工作区之前，必须先完成这一步
- 在 `BASE_ENTER_WORKSPACE` 阶段，手臂默认保持收回，不参与规划与补偿

为什么必须单独强调这一步：

- 底盘进工作区时不需要手臂参与
- 如果手臂伸在外面，容易碰到货架、柜口、周围物品
- 手臂外伸还会改变底盘通过空间，增加“底盘明明路线对但本体过不去”的风险
- 因此 V3 中应把“先收臂，再动车”作为硬规则，而不是经验操作

第一版临时收臂姿态建议：

- 先允许使用一组人工摆出来的 `transit_stow_v0`
- 它不是最终最优姿态，但可以先作为底盘运动前的统一安全模板
- 后续再通过碰撞检查和通行测试，把它优化成正式 `stow_safe` 模板

根据当前手摆示意图，可先记录一版近似模板：

- `raise_joint`: `0.302`
- `r1_joint`: `-39 deg`
- `r2_joint`: `-75 deg`
- `r3_joint`: `-75 deg`
- `r4_joint`: `-90 deg`
- `r5_joint`: `1 deg`
- `r6_joint`: `5 deg`

对应弧度近似值：

- `raise_joint`: `0.302`
- `r1_joint`: `-0.681`
- `r2_joint`: `-1.309`
- `r3_joint`: `-1.309`
- `r4_joint`: `-1.571`
- `r5_joint`: `0.017`
- `r6_joint`: `0.087`

V3 中建议把这组姿态命名为：

- `transit_stow_v0`

它的用途仅限于：

- 底盘进入工作区前的安全收臂
- 失败恢复时的临时回退姿态

它不承担：

- 柜口预对齐
- 柜外候选初始化
- 最终吸取前准备

### 6.3 `BASE_ENTER_WORKSPACE`

职责：

- 将底盘带到货架前一个可工作区域
- 不要求到达精确末端对齐位姿

输出：

- 一个“可接受的底盘工作区状态”

完成判据：

- 底盘位置 / 朝向进入宽松容差
- 或者人为确认底盘已经处于可操作区域

### 6.4 `LIFT_TO_BAND`

职责：

- 将升降杆调整到目标高度带
- 不是精确单点，而是一个允许手臂补偿的高度区间

完成判据：

- `raise_height` 进入 `[target_band_min, target_band_max]`

### 6.5 `SELECT_PRE_INSERT`

职责：

- 基于 `T_pick` 生成一组位于柜口外的候选 `T_pre_insert`
- 对候选进行轻量级预筛选和评分
- 选出最适合当前状态的候选

这是 V3 的核心新阶段。

### 6.6 `PLAN_TO_PRE_INSERT`

职责：

- 使用车臂联合规划到选中的 `T_pre_insert`
- 允许底盘、升降、机械臂共同参与，但规划目标位于柜口外

说明：

- 这一步解决“大范围接近”
- 不要求在柜内完成复杂避障

### 6.7 `INSERT_AND_SUCTION`

职责：

- 从 `T_pre_insert` 向 `T_pick` 做短距离插入
- 插入期间保持吸盘法向稳定
- 到位后触发吸附

规划策略：

- 首选短程 Cartesian 插入
- 失败时退化到强姿态约束的短程 OMPL
- 再失败则放弃当前候选，回到 `SELECT_PRE_INSERT`

### 6.8 `SAFE_RETREAT`

职责：

- 按与插入方向相反的安全方向撤出柜体
- 撤出后恢复到柜外安全位姿

说明：

- 该阶段优先采用直线撤出
- 不应一吸上就立刻做全局自由规划

### 6.9 `EXIT_OR_HANDOFF`

职责：

- 完成离柜、交接、或等待下一个任务

## 7. 关键设计：`prepare` 在 V3 中如何被重新定义

V3 中不再使用“prepare pose”这个带有旧语义包袱的说法，统一改称：

- `pre-insert pose`
- 中文可称“柜口外插入准备位姿”

它的定义不是“最终位姿沿法向回退一个固定值”，而是：

- 位于柜口外
- 姿态大体满足吸盘法向进入要求
- 横向偏差允许由最后短距离插入收敛
- 对底盘粗定位误差有一定容忍度
- 能够从当前状态稳定到达

也就是说，V3 的 `pre-insert` 是一个局部搜索问题，而不是单个公式解。

## 8. `pre-insert` 候选生成方案

### 8.1 生成原则

候选位姿必须满足：

- 在柜口外，不能先把中间位姿生成到柜内
- 与 `T_pick` 大体同向
- 允许少量横向偏置和少量偏航扰动
- 插入距离适中

### 8.2 候选参数化

建议第一版使用如下参数化：

- 回退距离 `d`
  - `[0.05, 0.08, 0.12, 0.15]`
- 横向偏置 `l`
  - `[-0.02, 0.0, 0.02]`
- 竖向微调 `h`
  - `[-0.01, 0.0, 0.01]`
- 绕柜体竖直方向的偏航扰动 `yaw`
  - `[-8°, -4°, 0°, 4°, 8°]`

第一版不建议把四组参数全部笛卡尔积展开到最大规模。建议使用两级候选策略：

- 粗候选：先只扫描 `d + l`
- 精候选：对粗候选前若干名再补 `h + yaw`

这样既保留了搜索能力，也避免一次性评估过多重候选。

### 8.3 几何生成公式

设：

- `T_pick` 为最终吸取位姿
- `n_entry` 为柜体进入方向
- `u_lat` 为与 `n_entry` 垂直的横向单位向量
- `u_up` 为世界或柜体定义的竖直方向单位向量

则候选 `T_pre_insert` 的位置可表示为：

`p_pre = p_pick - d * n_entry + l * u_lat + h * u_up`

姿态可表示为：

- 以 `T_pick` 为基准
- 绕 `u_up` 加小偏航扰动
- 必要时保留一个很小的吸盘滚转补偿量，但第一版可先不启用

## 9. 候选筛选与评分

### 9.1 轻量预筛选

候选不应一上来就全部做完整重规划。建议先做便宜检查：

1. 候选是否位于柜口外
2. 候选是否满足最基本姿态约束
3. 是否存在 IK 解
4. IK 解是否碰撞
5. 升降高度是否仍在可调带内

只有通过预筛选的候选，才进入重规划阶段。

### 9.2 重规划检查

对通过预筛选的候选，再做：

1. 从当前状态到 `T_pre_insert` 的联合规划
2. 从 `T_pre_insert` 到 `T_pick` 的短距离插入规划
3. 必要时评估撤出路径是否存在

### 9.3 评分函数

第一版评分建议只保留可靠、可解释的项：

`score = w1 * delta_q + w2 * approach_cost + w3 * insert_cost + w4 * align_error + w5 * robustness_penalty`

各项定义：

- `delta_q`
  - 从当前状态到候选 IK 解的关节变化量
- `approach_cost`
  - 到 `pre-insert` 的规划时间和轨迹长度
- `insert_cost`
  - 插入段长度、失败率或笛卡尔完成率代价
- `align_error`
  - 吸盘工具轴与目标法向的误差
- `robustness_penalty`
  - 对底盘误差和升降误差敏感的候选施加额外惩罚

第一版不建议强依赖“精确最小间隙”作为核心评分项，因为当前系统未必有稳定、低成本的 clearance 计算接口。可先把 clearance 作为 debug 指标，而不是主排序项。

## 10. V3 的规划策略

### 10.1 柜外阶段：联合自由规划

适用阶段：

- `PLAN_TO_PRE_INSERT`

策略：

- 使用 `mobile_arm` 或新的 `whole_body` 规划组
- 允许底盘、升降、机械臂联合规划到柜口外候选位姿
- 规划器优先 `RRTConnect`

原因：

- 柜外空间更开阔，采样规划效率更高
- 这段更关心“能到达”，而不是完全姿态刚性

### 10.2 柜口附近阶段：姿态受限规划

适用阶段：

- `PLAN_TO_PRE_INSERT` 的后半段
- `INSERT_AND_SUCTION` 的 fallback

策略：

- 对吸盘法向施加 orientation constraint
- 限制末端绕法向外的过大摆动

说明：

- 这一步是从“自由接近”过渡到“受限插入”
- 但第一版可不立即引入完整 constrained state space，仅先使用姿态约束 + 短路径规划

### 10.3 柜内阶段：短程 Cartesian 优先

适用阶段：

- `INSERT_AND_SUCTION`
- `SAFE_RETREAT`

策略：

- 优先做末端沿进入方向的短程 Cartesian 路径
- 如果笛卡尔完成率不足，再尝试短程约束 OMPL
- 若仍失败，则放弃当前候选

原因：

- 柜内动作的本质是插入 / 撤出，不是自由避障漫游
- 短程直进更符合吸盘抓取特性

## 11. 底盘与升降在 V3 中的角色重定义

### 11.1 底盘不再承担“精对齐”

V3 中底盘的职责是：

- 进入货架前工作区
- 提供手臂可操作的几何余量
- 让后续 `pre-insert` 候选有足够成功率

底盘不再承担：

- 将吸盘投影精确送到目标入口法平面上的某一点
- 为最终抓取提供毫米级横向对齐

补充原则：

- 底盘运动期间，默认机械臂处于 `transit_stow_v0` 或后续正式 `stow_safe` 姿态
- 只有在底盘已经进入工作区并稳定后，才允许进入候选 `pre-insert` 搜索与车臂联合规划

### 11.2 升降不再作为“连续笛卡尔维度”

V3 中升降职责是：

- 先把任务压缩到一个合适的高度带
- 通过高度回读形成闭环

升降不再承担：

- 柜内最终几厘米的精细插入补偿
- 与手臂一起进行复杂的末端微调插值

### 11.3 高度带设计

建议定义：

- `lift_target_center`
- `lift_target_tolerance`
- `lift_target_band_min`
- `lift_target_band_max`

其中：

- `center` 由目标药盒高度和经验标定映射得到
- `band` 由手臂剩余可补偿范围、吸盘长度、层板间隙共同决定

## 12. 末端吸盘特化设计

V3 必须把末端当作吸盘，而不是夹爪。

### 12.1 姿态目标

吸盘关键姿态不是“夹爪张口方向”，而是：

- 吸盘法向尽量对准目标表面法向
- 允许少量绕法向旋转自由度
- 不允许过大横摆

### 12.2 接触策略

插入末端的成功条件建议分层：

1. 位姿进入吸附允许窗
2. 插入深度达到要求
3. 姿态误差小于阈值
4. 触发吸附动作
5. 吸附确认成功后再撤出

### 12.3 撤出策略

撤出应先保持法向稳定，后离柜自由恢复。不要直接在柜内做大姿态调整。

## 13. V3 的场景建模要求

若场景模型不准确，V3 再好的候选搜索也会失真。

建议在 Planning Scene 中明确建模：

- 柜体外框
- 顶板 / 底板 / 层板
- 左右隔板
- 柜口厚度
- 药盒碰撞体
- 吸盘和手腕外形

额外要求：

- 吸盘碰撞不能删除
- 但需要避免明显错误的自碰撞干扰
- 柜口几何必须足够接近真实，否则 `pre-insert` 搜索会选出虚假好解

## 14. V3 代码架构设计

V3 应采用“几何 / 规划 / 控制 / 状态机”四层分离。

### 14.1 几何层

建议新增模块：

- `task_geometry.py`

职责：

- 解析目标位姿
- 计算 `entry_direction`
- 生成 `pre-insert` 候选
- 计算插入向量、回退距离、横向偏置

### 14.2 候选评估层

建议新增模块：

- `preinsert_selector.py`

职责：

- 候选预筛选
- IK 与碰撞快速检查
- 候选评分
- 输出最优 `PreInsertCandidate`

### 14.3 规划层

建议新增模块：

- `approach_planner.py`
- `insert_planner.py`
- `retreat_planner.py`

职责：

- `approach_planner`
  - 负责当前状态到 `pre-insert`
- `insert_planner`
  - 负责短程插入和吸附前推进
- `retreat_planner`
  - 负责撤出与离柜

### 14.4 执行层

建议新增模块：

- `base_controller_adapter.py`
- `lift_controller_adapter.py`
- `arm_executor.py`

职责：

- `base_controller_adapter`
  - 封装底盘粗到位、到位确认、误差容忍
- `lift_controller_adapter`
  - 封装升 / 降 / 停与高度回读闭环
- `arm_executor`
  - 封装 MoveIt 轨迹执行与结果确认

额外建议：

- 增加 `stow_manager.py`

职责：

- 管理 `transit_stow_v0`、正式 `stow_safe`、失败恢复收臂姿态
- 对底盘运动前是否已经收臂完成给出统一判断
- 为后续更换收臂模板留出独立接口

### 14.5 状态机层

保留一个轻量状态机模块：

- `pick_task_manager.py`

职责只剩：

- 维护阶段流转
- 调度各模块
- 处理超时 / 失败 / 重试 / 重置

V3 中状态机不再直接承担几何求解、候选打分和所有具体规划细节。

## 16. 失败恢复策略

V3 不应只有“成功 / 失败”两值，而应区分失败类别。

### 16.1 候选失败

表现：

- 某个 `pre-insert` 无 IK
- 碰撞
- approach 不可达
- insert 不可达

处理：

- 更换候选，不重启整个任务

### 16.2 柜外不可达

表现：

- 多个候选都无法到达

处理：

- 请求底盘重新进入工作区
- 或重新调整升降带

### 16.3 柜内插入失败

表现：

- approach 成功，但 insert 失败

处理：

- 保留当前柜外安全状态
- 换下一个 `pre-insert`
- 不必立即整机回到 stow

### 16.4 吸附失败

表现：

- 到位但未成功吸附

处理：

- 小幅撤回
- 允许重试若干次
- 超限后任务失败

## 19. 第一版 V3 的范围控制

为了避免 V3 再次失控，第一版建议明确不做以下事项：

- 不接入 MoveIt Task Constructor
- 不做大规模复杂候选并行求解
- 不把升降杆纳入柜内精细笛卡尔插入
- 不做过于复杂的最小间隙优化
- 不引入过多学习型策略

第一版 V3 的核心交付应是：

- 新阶段定义正确
- `pre-insert` 候选搜索可解释
- 插入动作是短程直进优先
- 底盘和升降的职责边界正确
- 吸盘逻辑是吸盘逻辑，而不是夹爪逻辑

## 20. 最终结论

V3 不是“把旧 `prepare` 调得更聪明”，而是重新定义任务：

- 底盘做粗工作区定位
- 升降做高度带对齐
- 候选搜索负责找到合适的柜口外 `pre-insert`
- 车臂联合规划负责到达 `pre-insert`
- Cartesian-first 插入负责完成柜内吸取
- 安全直退负责离柜

`v3初始.md` 提供了正确的方向，但本项目真正需要的是一版面向“底盘不准 + 升降离散 + 吸盘末端 + 货柜受限空间”的工程化 V3。本文档即作为该版本的正式设计基线。
